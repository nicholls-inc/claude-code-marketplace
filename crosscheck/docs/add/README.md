# `docs/add/`

This directory hosts the live work on assurance-driven development for the Crosscheck plugin. The first attempt (v1) was archived under `.retrospective/v1-archive/` after a retrospective concluded the surface was wrong. The v2 retrospective at `.retrospective/findings-and-methodology-v2.md` is the long-form record of what happened and the working hypothesis for v3.

Both validation runs the v3 shape was waiting on have now been driven: a mature-repo run (2026-05-15) and a greenfield run — `~/repos/ngst`, built spec-first and invariant-anchored from a near-empty repo, recorded in [`reports/add-greenfield-field-report-ngst.md`](reports/add-greenfield-field-report-ngst.md). The shape is validated; its automation is still mid-flight (the Phase-4 run-to-green agent, operating modes, and the Phase-5 auditor are tracked under epic [#217](https://github.com/nicholls-inc/claude-code-marketplace/issues/217)). For current state, read `JOURNAL.md` and the field reports. There is still no canonical methodology doc here, by design — it should follow the shipped automation, not lead it, and will be written to describe what shipped once the #217 children land.
